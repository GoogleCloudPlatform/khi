KHI uses Common Expression Language (CEL) for advanced, multi-stage filtering.

1. Timeline Filter (Include / Exclude)

Filters which resource timelines to display (e.g., by namespace, kind, or name). Supports both Include filters (showing matching timelines and their descendants) and Exclusion filters (hiding matching timelines and their descendants).

2. Log Filter

Filters individual logs (events/revisions) inside the active timelines.

Tip: Timelines are filtered first, then KHI filters the logs within them. Filtering logs without a timeline filter is extremely slow because KHI must scan all logs. Always narrow down timelines first for better performance.

CEL allows complex filtering that the standard UI cannot configure, such as:

  • Matching logs by nested JSON fields (e.g., finding logs associated with a specific operation.id).
  • Timelines where a specific manifest field has a specific value (e.g., pods scheduled on a specific node via spec.nodeName).
