Timeline CEL expressions filter which resource timelines are shown or excluded. The expression must evaluate to a boolean.

Timeline Exclusion Filter

When an expression is used in the Timeline Exclusion filter, any timeline that evaluates to true and all of its descendant timelines (e.g., Pods belonging to an excluded Node or Workload) will be hidden from the view.

Examples

Filter by exact value across any timeline type match('^kube-system$')
Filter by regex pattern across any timeline type match('^kube-.*')
Filter by substring across any timeline type match('kube-')
Filter by namespace timeline starting with 'kube-' using regex match('namespace', '^kube-.*')
Filter timelines containing revisions with a specific Pod IP revision_body('status.podIP', '10.0.0.1')
Filter timelines containing revisions with Pod IP starting with '10.' using regex revision_body('status.podIP', '^10\\..*')
Exclude kube-system namespace and all resources inside it (for Exclusion filter) match('namespace', '^kube-system$')

Helper Functions

Function Description
match(value)
M(value)		 Checks if any key in the path map contains the given value or regex pattern case-insensitively.
match(key, value)
M(key, value) 		Checks if the specified path property key contains the given value or regex pattern case-insensitively.
revision_body(key, value)
RB(key, value) 		Checks if any revision within the timeline matches the specified dot-separated body path and pattern.

Variables Available

Variable Type Description
name string The name of the resource timeline (e.g. pod name).
timelineType string The type/kind of the timeline (e.g. pod, node).
path map Path properties map (e.g. namespace, apiVersion, kind).
